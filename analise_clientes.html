Case técnico Jeitto¶

Este notebook contem anlises de uma base de dados sobre clientes e suas compras. Para guia este estudo iremos levantar as seguintes questes a respeito dos clientes:

  • Na primeira compra os clientes tendem a comprar qual produto?
  • Qual a distribuio dos clientes por estado?
  • Qual a distribuio dos clientes por faixas etrias?
  • Qual o ticket medio dos clientes?
  • Qual o produto mais vendido por regiao do pas (sul, sudeste, norte, nordeste, centro-oeste) ?
  • Quantos clientes fantasmas (nunca compraram nada) existem na base?

def ageRange(age):
    minAge = 19
    maxAge = minAge + 5
    ageRangeClass = 1
    while True:
        if age >= minAge and age <= maxAge:
            return str(minAge) + "-" + str(maxAge)
        else:
            ageRangeClass += 1
            minAge = maxAge 
            maxAge = minAge + 5

ghostCustomerOfAge["age_range"] = ghostCustomerOfAge["age"].apply(ageRange)

regions = dict()

regions["southeast"] = ["SP", "RJ", "MG", "ES"]
regions["south"] = ["RS", "SC", "PR"]
regions["nort"] = ["TO", "AC", "PA", "RO", "RR", "AP" "AM"]
regions["northeast"] = ["BA", "SE", "AL", "PB", "PE", "RN", "CE", "PI", "MA"]
regions["midwest"] = ["MT", "MS", "GO", "DF"]
In [17]:
def defineCountryRegion(state):
    for reg in regions.keys():
        if state in regions[reg]:
            return reg
    return np.nan

Questões teóricas¶

Conceitos sobre Bancos de dados

Atualmente existem diversos tipos de bancos de dados disponveis no mercado, muitos deles open source e com caractersticas de funcionamento e aplicao diferentes entre si. Descreva as principais caractersticas e diferenas entre os seguintes tipos de bancos de dados:

  • Bancos de dados relacionais: os dados so organizados em tabelas. A maneira como os dados so organizados possibilitam acessar dados que estejam relacionados entre si (ex: um pedido de pizza e um cliente devem estar relacionados. O pedido de pizza deve conter um ID que possibilite encontrar qual o cliente responsvel pelo mesmo). H diversos SGBD disponveis no mercado, como MySQL, SQLite e PostegreSQL.

Fonte: Codeacademy.

  • Bancos de dados no relacionais: um banco de dados que no segue o modelo relacional. Em bancos NoSQL a arquitetura do banco pode ser variada mais facilmente, enquanto que em bancos relacionais ela precisa ser definida antes da implementao. Desta forma, o banco no relacional permite a insero de dados sem um esquema pr definido. A vantagem de se utilizar esse tipo de banco que alteraes na arquitetura do banco so mais fceis de serem realizadas do que em bancos relacionais. Fonte: MongoDB.

  • Bancos de dados de grafos: novamente, este tipo de banco armazena os dados sem um esquema pr-definido. Os dados so armazenados em uma estrutura de grafo e esto conectados entre si. Cada n do grafo armazena os dados em si. (ex: nome, data de nascimento e etc). A conexo (aresta) entre dois ns, representa a relao entre eles. E este relacionamento tambm pode ter propriedades a seu respeito. Fonte NEO4j.

Manipulao de dados com SQL

Voc possui acesso as tabelas de um banco relacional conforme o esquema abaixo de colunas em cada tabela (cada caixa uma tabela com os nomes das colunas disponveis).

Construa as consultas SQL para responder as seguintes questes (voc pode utilizar a sintaxe SQL que preferir):

  • Gere uma relao dos clientes da empresa com nome, uf e e-mail ordenados do mais recente para o mais antigo
    SELECT cp.(nome, uf, email) FROM cadastro_perfil cp, cadastro_sistema cs
WHERE cp.id_usuario = cs.id_usuario
ORDER BY cs.data_cadastro DESC

  • Construa uma consulta que retorne os produtos mais vendidos com nome do produto e quantidade de venda

    SELECT p.nome_produto, COUNT(p.nome_produto) FROM table produtos p, vendas v
WHERE p.id_produto = v.id_produto
GROUP BY p.nome_produto
ORDER BY COUNT(*) DESC

  • Agrupe a consulta anterior por meses do ano

    SELECT p.nome_produto, COUNT(p.nome_produto) FROM table produtos p, vendas v
WHERE p.id_produto = v.id_produto
GROUP BY p.nome_produto, MONTH(v.data_venda)
ORDER BY COUNT(*) DESC
