Case técnico Jeitto¶

Este notebook contem análises de uma base de dados sobre clientes e suas compras. Para guia este estudo iremos levantar as seguintes questões a respeito dos clientes:

  • Na primeira compra os clientes tendem a comprar qual produto?
  • Qual a distribuição dos clientes por estado?
  • Qual a distribuição dos clientes por faixas etárias?
  • Qual o ticket medio dos clientes?
  • Qual o produto mais vendido por regiao do país (sul, sudeste, norte, nordeste, centro-oeste) ?
  • Quantos clientes fantasmas (nunca compraram nada) existem na base?

def ageRange(age):
    minAge = 19
    maxAge = minAge + 5
    ageRangeClass = 1
    while True:
        if age >= minAge and age <= maxAge:
            return str(minAge) + "-" + str(maxAge)
        else:
            ageRangeClass += 1
            minAge = maxAge 
            maxAge = minAge + 5

ghostCustomerOfAge["age_range"] = ghostCustomerOfAge["age"].apply(ageRange)

regions = dict()

regions["southeast"] = ["SP", "RJ", "MG", "ES"]
regions["south"] = ["RS", "SC", "PR"]
regions["nort"] = ["TO", "AC", "PA", "RO", "RR", "AP" "AM"]
regions["northeast"] = ["BA", "SE", "AL", "PB", "PE", "RN", "CE", "PI", "MA"]
regions["midwest"] = ["MT", "MS", "GO", "DF"]
In [17]:
def defineCountryRegion(state):
    for reg in regions.keys():
        if state in regions[reg]:
            return reg
    return np.nan

Questões teóricas¶

Conceitos sobre Bancos de dados

Atualmente existem diversos tipos de bancos de dados disponíveis no mercado, muitos deles open source e com características de funcionamento e aplicação diferentes entre si. Descreva as principais características e diferenças entre os seguintes tipos de bancos de dados:

  • Bancos de dados relacionais: os dados são organizados em tabelas. A maneira como os dados são organizados possibilitam acessar dados que estejam relacionados entre si (ex: um pedido de pizza e um cliente devem estar relacionados. O pedido de pizza deve conter um ID que possibilite encontrar qual o cliente responsável pelo mesmo). Há diversos SGBD disponíveis no mercado, como MySQL, SQLite e PostegreSQL.

Fonte: Codeacademy.

  • Bancos de dados não relacionais: é um banco de dados que não segue o modelo relacional. Em bancos NoSQL a arquitetura do banco pode ser variada mais facilmente, enquanto que em bancos relacionais ela precisa ser definida antes da implementação. Desta forma, o banco não relacional permite a inserção de dados sem um esquema pré definido. A vantagem de se utilizar esse tipo de banco é que alterações na arquitetura do banco são mais fáceis de serem realizadas do que em bancos relacionais. Fonte: MongoDB.

  • Bancos de dados de grafos: novamente, este tipo de banco armazena os dados sem um esquema pré-definido. Os dados são armazenados em uma estrutura de grafo e estão conectados entre si. Cada nó do grafo armazena os dados em si. (ex: nome, data de nascimento e etc). A conexão (aresta) entre dois nós, representa a relação entre eles. E este relacionamento também pode ter propriedades a seu respeito. Fonte NEO4j.

Manipulação de dados com SQL

Você possui acesso as tabelas de um banco relacional conforme o esquema abaixo de colunas em cada tabela (cada caixa é uma tabela com os nomes das colunas disponíveis).

Construa as consultas SQL para responder as seguintes questões (você pode utilizar a sintaxe SQL que preferir):

  • Gere uma relação dos clientes da empresa com nome, uf e e-mail ordenados do mais recente para o mais antigo
    SELECT cp.(nome, uf, email) FROM cadastro_perfil cp, cadastro_sistema cs
WHERE cp.id_usuario = cs.id_usuario
ORDER BY cs.data_cadastro DESC

  • Construa uma consulta que retorne os produtos mais vendidos com nome do produto e quantidade de venda

    SELECT p.nome_produto, COUNT(p.nome_produto) FROM table produtos p, vendas v
WHERE p.id_produto = v.id_produto
GROUP BY p.nome_produto
ORDER BY COUNT(*) DESC

  • Agrupe a consulta anterior por meses do ano

    SELECT p.nome_produto, COUNT(p.nome_produto) FROM table produtos p, vendas v
WHERE p.id_produto = v.id_produto
GROUP BY p.nome_produto, MONTH(v.data_venda)
ORDER BY COUNT(*) DESC

Para que as consultas que você construiu na questão anteriores sejam executadas com performance você precisará que as tabelas estejam corretamente indexadas. Considerando isso responda as seguintes questões:

  • O que é o índice em uma tabela de banco de dados relacional? Qual a sua necessidade? Um indíce numa tabela é um valor que identifica um registro de forma única. Ele é útil quando é necessário recuperar um registro de forma eficiente. Sabendo o índice do registro, é mais fácil encontrá-lo bastando procurar pelo valor do índice. Fonte: DevMedia.
  • Quais índices deveriam ser criados para acelerar as consultas da questão anterior? Para acelerar as consultas, poderiam ser criados indices nos campos id.
